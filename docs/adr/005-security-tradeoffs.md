# ADR-005: 보안 트레이드오프

## 상태
승인 (데모 환경 한정)

## 맥락
데모 앱에서 Claude API를 호출해야 하지만, 방산 환경의 보안 요구사항도 인지하고 있음을 보여야 한다.

## 결정
개발/데모 환경과 프로덕션 환경을 분리하는 이중 구조를 채택한다.

### 데모 환경 (현재)
- 브라우저에서 Claude API 직접 호출 (anthropic-dangerous-direct-browser-access)
- API 키는 localStorage에 저장 (사용자 입력)
- CSP 메타 태그로 기본 보호
- 코드 주석으로 "DEMO ONLY" 명시

### 프로덕션 설계 (향후)
- 서버 사이드 프록시: Vercel 서버리스 함수에서 API 호출 (구현 완료)
- API 키: 환경 변수로 관리, 브라우저 노출 없음
- 온프레미스 LLM: 사내망 전용, 외부 통신 차단
- 입력 검증: 프롬프트 인젝션 방어 (길이 제한 + 패턴 필터링)
- 감사 로그: 모든 AI 호출 기록 (군사 보안 감사 대응)

## 인지하는 위험
1. API 키 브라우저 노출 → 프록시로 해결 (이미 구현)
2. 프롬프트 인젝션 → 입력 검증 계층 추가 필요
3. LLM 환각(Hallucination) → 생성 결과는 반드시 개발자 검토 후 반영
4. 군사 기밀 데이터 → 클라우드 AI 사용 불가, 온프레미스 필수

## 결과
- 면접관에게: "보안 리스크를 인지하고, 각각에 대한 대응 방안을 설계한 개발자"
- 코드에: 모든 보안 관련 결정에 주석으로 근거 명시
